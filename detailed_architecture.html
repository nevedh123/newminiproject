UNIO Platform - Detailed Architecture

1. API Routing & Security Architecture

HTTP Header: Authorization: Bearer <JWT_TOKEN>
Authentication (/api/auth)
POST /register
Expects {name, email, password, role}
POST /login
Expects {email, password}
-> Returns JWT
GET /me 🔒
Protected (All Roles)
-> Fetches current user info
Listings Management (/api/listings)
GET /listings 🌐
Publicly fetches all approved listings
POST /listings 🔒
Protected (Provider)
-> Expects {type, capacity, price, location}
GET /listings/{id} 🌐
Publicly fetches details of a single listing
PUT /listings/{id} 🔒
Protected (Provider)
-> Updates existing listing payload
Bookings Management (/api/bookings)
POST /bookings 🔒
Protected (Consumer)
-> Expects {listing_id, quantity}
GET /bookings 🔒
Protected (Consumer)
-> Fetches historical consumer bookings
PUT /bookings/{id}/status 🔒
Protected (Admin/System)
-> Confirms or cancels booking
Community Splits (/api/splits)
GET /splits 🔒
Protected (Consumer)
Fetches open community room splits
POST /splits/{id}/join 🔒
Adds user to split & calculates fractional share
Payment Simulator (/api/payments)
POST /initiate 🔒
Generates internal IP dynamic QR Code
GET /scan/{payId}/{confId} 🌐
Simulates mobile device scanning & approval
Admin & Analytics (/api/admin)
GET /stats/dashboard 🔒
Protected (Admin)
-> Fetches total marketplace metrics
PUT /listings/{id}/status 🔒
Protected (Admin)
-> Approves/Rejects pending resources
